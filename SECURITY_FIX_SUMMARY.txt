================================================================================
HOW THE FIX WORKS
================================================================================

VULNERABLE APPROACH:
  const userInput = process.argv[2];          // e.g., "; rm -rf /"
  execSync(`npm install ${userInput}`);
  // Shell executes: npm install ; rm -rf /
  // Result: TWO commands run - both dangerous!

SECURE APPROACH:
  const userInput = process.argv[2];          // e.g., "; rm -rf /"
  validateThemeName(userInput);               // Throws error!
  execFileSync('npm', ['install', userInput]);
  // npm receives: npm install ""; rm -rf /"
  // Result: Package name is literal string, not command

KEY DIFFERENCES:
  • Validation BEFORE execution (defense in depth)
  • execFileSync bypasses shell entirely
  • Arguments are literal strings (no metacharacter interpretation)
  • Shell cannot inject commands into argument arrays
